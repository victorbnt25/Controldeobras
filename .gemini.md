# PROYECTO: DECOREFORM.A.B

## 🧠 IDEA PRINCIPAL

Aplicación web interna para gestión empresarial de una empresa real de reformas.

El sistema permite:

- Gestión de clientes
- Gestión de trabajadores
- Gestión de proveedores
- Creación y gestión de presupuestos
- Conversión automática de presupuestos aceptados en obras
- Control económico en tiempo real de cada obra
- Gestión futura de facturas

El objetivo principal es tener control financiero claro y visual para el dueño de la empresa.

# 🗄 ARQUITECTURA BASE DE DATOS (MySQL)

## TABLAS PRINCIPALES

### clientes
- id
- nombre
- tipo (empresa / particular)
- cif_dni
- direccion
- telefono
- email
- activo

---

### presupuestos
- id
- numero_presupuesto
- fecha_presupuesto
- cliente_id (FK)
- descripcion_general
- total_bruto
- iva_porcentaje
- iva_importe
- total_presupuesto
- estado (pendiente / aceptado / rechazado)
- creado_factura

---

### presupuesto_items
- id
- presupuesto_id (FK)
- descripcion
- cantidad
- precio_unitario
- importe (GENERATED)

---

### obras
- id
- presupuesto_id (UNIQUE FK)
- cliente_id (FK)
- numero_obra
- titulo
- descripcion
- presupuesto_total (FIJO)
- gastado (se actualiza dinámicamente)
- restante (calculado)
- created_at

Una obra solo puede existir si el presupuesto fue aceptado.

---

### gastos
- id
- obra_id (FK)
- concepto
- importe
- fecha

Los gastos afectan al cálculo de:
- Total gastado
- Presupuesto restante

---

# 🔄 FLUJO DE NEGOCIO

1. Se crea presupuesto
2. Presupuesto tiene estado = pendiente
3. Si se acepta:
   - Se cambia estado a aceptado
   - Se crea automáticamente una obra
   - La obra hereda el presupuesto_total
4. En la obra:
   - Se pueden añadir gastos
   - Se recalcula gastado
   - Se recalcula restante
   - El presupuesto_total NUNCA cambia
